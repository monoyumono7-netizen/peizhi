# Markdown 输出管理器

统一管理所有 markdown 文件的输出位置和命名规范。

## 基础路径

```
/Users/mono/Desktop/markdown/
```

## 输出规则

### 1. 确定输出文件夹名

按输出模式区分命名策略：

#### 扁平模式（单文件输出）

按以下优先级自动确定：
1. **Git 仓库名**（优先）：`basename $(git rev-parse --show-toplevel 2>/dev/null)`
2. **当前工作目录名**（如果不是 git 仓库）
3. **对话上下文推断**（如果用户提到了具体项目）

#### 分层目录模式（多文档输出）

由 AI 根据分析内容自主命名，不需要向用户确认。

**命名指引：**
- 包含**项目归属 + 分析主题**，如 `cocraft-AI-Fusion聊天框`、`CBChat-组件库`
- 如果分析的是整个项目，直接用项目名，如 `cocraft`
- 避免过长，控制在 20 字以内
- 使用中文描述主题部分

**文件前缀（`{前缀}`）：** 取文件夹名的简短形式，用于文件夹内所有文件的统一前缀。通常是项目名或分析主题的缩写。

**示例：**
| 分析范围 | 文件夹名 | 文件前缀 |
|---------|---------|---------|
| cocraft 的 AI 助手模块 | `cocraft-AI-Fusion聊天框` | `cocraft` |
| CBChat 组件库整体 | `CBChat-组件库` | `CBChat` |
| 某个 npm 包 | `cloud-agent-SDK` | `cloud-agent` |
| 整个项目 | `cocraft` | `cocraft` |

### 2. 输出路径规则

#### 扁平模式（默认）

单个文档输出时使用：

```
/Users/mono/Desktop/markdown/{文件夹名}/{前缀}-{描述}.md
```

示例：
- `/Users/mono/Desktop/markdown/cocraft/cocraft-AI助手架构分析.md`
- `/Users/mono/Desktop/markdown/CBChat/CBChat-组件源码解读.md`

#### 分层目录模式

当分析内容包含**多个模块、多个流程或多个设计模式**时，使用分层目录：

```
/Users/mono/Desktop/markdown/{文件夹名}/
├── {前缀}-{分析主题}-架构总览.md       # 架构总览文档（必须）
├── 模块/                                # 功能模块子文件夹
│   ├── {前缀}-{模块名}-详细分析.md
│   └── ...
├── 流程/                                # 核心流程子文件夹
│   ├── {前缀}-{流程名}-详细分析.md
│   └── ...
└── 设计模式/                            # 设计模式子文件夹（按需创建）
    └── ...
```

`{文件夹名}` 和 `{前缀}` 由 AI 根据分析内容自主确定（见上方"确定输出文件夹名"章节）。

**何时使用分层目录：**
- 分析涉及 ≥2 个功能模块或 ≥2 个流程
- 需要输出 ≥3 个 md 文件
- 用户明确要求"详细分析各模块/流程"

**分层示例**（分析 cocraft 项目的 AI Fusion 聊天框）：
```
/Users/mono/Desktop/markdown/cocraft-AI-Fusion聊天框/
├── cocraft-AI-Fusion聊天框-架构总览.md
├── 模块/
│   ├── cocraft-入口与容器层-详细分析.md
│   ├── cocraft-Hooks层-详细分析.md
│   └── cocraft-状态管理层-详细分析.md
└── 流程/
    ├── cocraft-消息发送流程-详细分析.md
    └── cocraft-会话管理流程-详细分析.md
```

**子文件夹命名规则：**
- 使用中文分类名：`模块/`、`流程/`、`设计模式/`
- 如果只有一种类型的详细分析，不创建子文件夹，直接放文件夹下

### 3. 文件命名规范

**格式**: `{前缀}-{描述性名称}.md`

`{前缀}` 由 AI 根据场景自主确定（扁平模式下通常为 Git 仓库名，分层目录下为文件夹名的简短形式）。

**描述性名称要求**:
- 使用中文
- 简洁明了，2-6 个词
- 反映文件核心内容
- 避免使用日期（除非是日志类文件）

**命名示例**:
| 内容类型 | 命名示例 |
|---------|---------|
| 代码分析 | `cocraft-AI助手架构分析.md` |
| 功能说明 | `cocraft-状态管理模式.md` |
| 流程文档 | `cocraft-登录流程梳理.md` |
| 问题排查 | `cocraft-虚拟列表闪烁问题.md` |
| 代码审查 | `cocraft-PR123审查报告.md` |

### 4. 目录管理

- **自动创建**: 如果项目文件夹不存在，自动创建
- **文件冲突**: 如果同名文件已存在，询问用户是否覆盖或追加序号

## 工作流程

### 步骤 1：确定文件夹名和文件前缀

- **扁平模式**：优先取 Git 仓库名，其次取当前目录名
- **分层目录模式**：AI 根据分析内容自主命名（见"确定输出文件夹名"章节）

### 步骤 2：检查/创建目录

```bash
mkdir -p "/Users/mono/Desktop/markdown/${FOLDER_NAME}"
```

### 步骤 3：生成文件名

根据输出内容，生成简洁的描述性名称。

### 步骤 4：输出文件

将内容写入到：
```
/Users/mono/Desktop/markdown/{文件夹名}/{前缀}-{描述}.md
```

### 步骤 5：返回结果

告知用户：
- 文件保存路径
- 文件大小
- 可用 `open` 命令打开

## 使用示例

**用户输入**: "把刚才的分析输出到md"

**执行过程**:
1. 检测当前在 `/Users/mono/cocraft` 目录
2. Git 仓库名为 `cocraft`
3. 确认目录 `/Users/mono/Desktop/markdown/cocraft/` 存在（不存在则创建）
4. 根据内容生成名称：`AI助手状态管理`
5. 输出到 `/Users/mono/Desktop/markdown/cocraft/cocraft-AI助手状态管理.md`
6. 返回："已保存到 /Users/mono/Desktop/markdown/cocraft/cocraft-AI助手状态管理.md"

## 其他 skill 集成

当其他 skill（如 code-analyzer）需要输出 markdown 文件时，应遵循本规则：

1. 根据输出模式确定文件夹名和文件前缀
2. 根据文档数量选择扁平模式或分层目录模式
3. 遵循文件命名规范
